# Gemini Research (with Claude Verification)

Gemini CLI를 활용하여 기술 조사를 수행하고, **Claude가 결과를 검증/보완**하는 스킬입니다.

## 핵심 변경사항 (v4.0.0)

1. **모델 선택**: 기본 `gemini-3-pro-preview` 사용, 실패 시 `gemini-3-flash-preview`로 자동 전환
2. **Claude 검증 단계**: Gemini 조사 결과를 그대로 전달하지 않고, Claude가 정확성 검증 및 누락 정보 보완
3. **신뢰도 향상**: 잘못된 정보나 오래된 정보 필터링
4. **메인 컨텍스트 실행**: 1~2회 호출로 끝나므로 Subagent 오버헤드 불필요
5. **[NEW] 기존 조사 캐싱**: 유사 주제 조사 존재 시 재활용 여부 확인
6. **[NEW] 출처 신뢰도**: 공식문서/블로그/기타로 출처 분류
7. **[NEW] 시각적 UI**: 화려한 배너, 진행률 바, 상태 아이콘으로 진행 상황 표시
8. **[NEW] 크로스-스킬 컨텍스트**: `.gemini/context.md` 연동

## Gemini 모델 선택 규칙

### 기본 모델
- **Primary**: `gemini-3-pro-preview` (기본값)
- **Fallback**: `gemini-3-flash-preview` (실패 시 자동 전환)

### 호출 방식
```bash
# 1차 시도: gemini-3-pro-preview
cat {파일} | gemini -m gemini-3-pro-preview -p "{프롬프트} ultrathink" -o json

# 실패 시 2차 시도: gemini-3-flash-preview
cat {파일} | gemini -m gemini-3-flash-preview -p "{프롬프트} ultrathink" -o json
```

### 프롬프트 규칙
- 모든 Gemini 프롬프트의 마지막에 **"ultrathink"** 키워드를 반드시 포함
- **웹 검색 지시**: 프롬프트에 "웹 검색을 통해 최신 정보를 확인하고, 출처를 함께 제시해주세요."를 포함

### Fallback 판단 기준
- exit code가 0이 아닌 경우
- JSON 파싱 실패
- `response` 필드가 비어있는 경우
- 타임아웃 발생

### Fallback 발생 시 사용자 알림
모델 변경 시 **반드시** 사용자에게 알립니다:
```
⚠️ 모델 변경 알림: gemini-3-pro-preview 호출 실패로 gemini-3-flash-preview로 전환하여 진행합니다.
```

## 작업 산출물 저장 규칙

모든 파일은 **프로젝트 디렉토리 내** `.gemini/` 폴더에 저장합니다.

### 저장 경로 구조
```
{프로젝트루트}/.gemini/research/{YYYYMMDD_HHMMSS}_{주제요약}/
  ├── research_input.txt              (초기 조사 질문)
  ├── research_gemini.md              (Gemini 조사 결과 원본)
  ├── research_claude_verification.md (Claude 검증 - 확인/수정/보완)
  ├── research_followup.txt           (후속 질문, 있을 경우)
  ├── research_followup_gemini.md     (후속 조사 결과, 있을 경우)
  └── research_final.md               (최종 검증된 결과)
```

## Gemini CLI 호출 규칙

### 모델 지정 (필수)
- 모든 Gemini 호출에 `-m gemini-3-pro-preview` 옵션 포함
- 실패 시 `-m gemini-3-flash-preview`로 재시도

### 입력 전달
- 코드나 긴 텍스트는 반드시 임시 파일로 작성 후 `cat` 또는 `<`로 전달
- 지시사항은 `-p` 플래그로 전달
- `echo "내용" | gemini` 패턴은 셸 이스케이핑 문제로 **사용 금지**

### 출력 처리
- 항상 `-o json` 플래그 사용
- `response` 필드에서 Gemini 답변 추출
- `session_id` 필드 보존 (후속 호출용)

### 세션 유지
- 후속 질문이 필요한 경우 `--resume latest` 사용
- 첫 호출에서는 `--resume` 미사용

### 에러 대응
1. **1차**: `gemini-3-pro-preview`로 시도
2. **2차**: 실패 시 `gemini-3-flash-preview`로 재시도 + 사용자 알림
3. **3차**: 둘 다 실패 시 Claude 단독으로 조사 수행

### Step 0: 기존 조사 확인 [NEW]

1. 구분선 및 진행 상황 출력:
```

  🎯 STEP 0: CHECK EXISTING RESEARCH
═══════════════════════════════════════════════════════════════

└────────────────────────────────────────────────────────────┘
🔄 Step 0: 기존 조사 확인 중...
```

2. Glob 도구로 `.gemini/research/` 폴더 검색
3. 유사 주제 존재 시 AskUserQuestion 호출:
```
이전에 유사한 조사가 있습니다:
- 2026-02-03: "Python 비동기 처리" (.gemini/research/20260203_...)

어떻게 진행할까요?
[1] 새로 조사 (기존 무시)
[2] 기존 결과 활용 (조사 생략)
[3] 기존 + 추가 조사 (보완)
```

4. 선택에 따라:
   - [1]: 새 조사 진행
   - [2]: 기존 `research_final.md` 반환
   - [3]: 기존 결과 참조하며 추가 조사

### Step 1: 조사 질문 구성 및 Gemini 호출

1. Bash로 `.gemini/research/{타임스탬프}_{주제}` 폴더 생성

2. Write 도구로 질문을 저장:
   - 경로: `{프로젝트루트}/.gemini/research/{타임스탬프}_{주제}/research_input.txt`
   - 내용: 구체적인 조사 질문

3. Gemini CLI 호출 (1차 시도):
```bash
cat {경로}/research_input.txt | gemini -m gemini-3-pro-preview -p "다음 항목을 포함해서 답변해주세요: 1. 개요 및 핵심 개념 2. 주요 방법론/접근법 3. 장단점 비교 4. 추천 라이브러리/도구 5. 코드 예시 (간단히). 웹 검색을 통해 최신 정보를 확인하고, 출처를 함께 제시해주세요. ultrathink" -o json
```

4. 실패 시 2차 시도 + 사용자 알림:
```bash
cat {경로}/research_input.txt | gemini -m gemini-3-flash-preview -p "다음 항목을 포함해서 답변해주세요: 1. 개요 및 핵심 개념 2. 주요 방법론/접근법 3. 장단점 비교 4. 추천 라이브러리/도구 5. 코드 예시 (간단히). 웹 검색을 통해 최신 정보를 확인하고, 출처를 함께 제시해주세요. ultrathink" -o json
```
```
⚠️ 모델 변경 알림: gemini-3-pro-preview 호출 실패로 gemini-3-flash-preview로 전환하여 진행합니다.
```

5. 응답 처리:
   - JSON 응답에서 `response` 필드 추출
   - `session_id` 필드 보존 (후속 질문용)
   - `response` 내용을 `research_gemini.md`로 저장

### Step 2: Claude 검증 (핵심 단계)

Gemini 조사 결과의 각 항목에 대해 Claude가 검증:

**검증 기준:**
1. **정확성**: 기술적으로 맞는 정보인가?
2. **최신성**: 오래된 정보나 deprecated된 내용은 없는가?
3. **완전성**: 중요한 내용이 누락되지 않았는가?
4. **맥락 적합성**: 사용자의 상황에 적합한 정보인가?

**검증 결과 분류:**

| 분류 | 설명 | 조치 |
|------|------|------|
| ✅ 확인 | 정확한 정보 | 그대로 최종 결과에 포함 |
| 🔧 수정 | 부정확/오래된 정보 | 올바른 정보로 교체 |
| ➕ 보완 | 누락된 중요 정보 | Claude가 추가 |
| ⚠️ 주의 | 검증 불가 | 주의 표시와 함께 포함 |

**출처 신뢰도 분류 [NEW]:**

| 분류 | 설명 | 표시 |
|------|------|------|
| 📚 공식문서 | 공식 문서, RFC, 표준 | 높은 신뢰도 |
| 📝 블로그/튜토리얼 | 기술 블로그, 개인 글 | 중간 신뢰도 |
| ❓ 기타/불명 | 출처 불분명 | 검증 필요 |

검증 결과를 `research_claude_verification.md`에 저장:
```markdown
## Claude 검증 결과

### ✅ 확인됨 (정확한 정보)
1. **[Gemini 내용 1]**
   - Claude 확인: 정확함.

### 🔧 수정 필요
1. **[Gemini 내용 2]**
   - Gemini 원본: "라이브러리 X는 Y 방식으로 동작"
   - Claude 수정: "라이브러리 X는 v2.0부터 Z 방식으로 변경됨"
   - 근거: 공식 문서 / Claude 지식

### ➕ 보완 (누락된 정보)
1. **[추가 정보]**
   - Claude 보완: Gemini가 언급하지 않았지만 중요한 내용...

### ⚠️ 주의 (검증 불가)
1. **[Gemini 내용 3]**
   - Claude 의견: 이 정보는 검증이 어려움. 공식 문서 확인 권장.
```

### Step 3: 후속 조사 (필요시)

누락된 정보가 많거나 심화가 필요한 경우:

1. Write 도구로 후속 질문을 저장:
   - 경로: `{경로}/research_followup.txt`
   - 내용: 구체적인 후속 질문

2. Gemini CLI 호출 (1차 시도):
```bash

```

```bash

```

4. 응답 → `research_followup_gemini.md` 저장
5. Claude 추가 검증

## Claude Code와의 연계

조사 완료 후:
1. 사용자가 구현 요청 시 → Claude Code가 **검증된** 조사 결과 기반으로 코드 작성
2. 추가 논의 필요 시 → `gemini-discuss` 스킬 활용
3. 코드 협업 필요 시 → `gemini-collab` 스킬 활용

## 에러 발생 시 대체 처리

1. **1차 실패** (`gemini-3-pro-preview`): `gemini-3-flash-preview`로 재시도 + 사용자 알림
2. **2차 실패** (`gemini-3-flash-preview`): Claude가 단독으로 조사 수행
3. **파일 작성 실패**: 사용자에게 알리고 대체 방안 모색

## 주의사항

- 조사 결과는 AI 지식 기반이며, 최신 정보가 아닐 수 있음
- Claude 검증도 완벽하지 않으므로 중요한 결정은 공식 문서로 재확인 권장
- 조사 범위가 넓으면 여러 번 나눠서 질문
- 검증 불가 항목은 반드시 주의 표시
- **모델 Fallback 발생 시 반드시 사용자에게 알림**
